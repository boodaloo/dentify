## Текущая структура (screenshot0701.png)

```

│ DentApp             │
├─────────────────────┤
│ 🏠 Dashboard        │ ← активный

│ 📋 Records          │
```

**Проблемы:**
- Не хватает модулей: Склад, Аналитика, Филиалы

- Billing слишком общий (счета/оплаты/баланс)
- Нет разделения по ролям

**Список табов на странице настроек:**

| Таб | Доступ | Описание |

| 📋 Клиника | Все | Название, логотип, региональные настройки, локальные папки |
| 🏢 Филиалы | Owner, Admin | Список филиалов, адреса, кабинеты, цветовая маркировка |
| 👥 Сотрудники | Owner, Admin | Врачи, администраторы, роли, специализации, график |
| 💊 Услуги | Owner, Admin | Категории услуг, прайс-лист, цены по филиалам |

| 📄 Документы | Owner, Admin | Шаблоны документов, пакеты, настройки печати |

| 💳 Подписка | **Owner only** | Тариф, история платежей, способы оплаты |
| 🔌 Доп. возможности | **Owner only** | Feature flags, платные add-ons, управление доступом |

**Ключевые принципы навигации:**

✅ **Sidebar (главное меню) всегда видимый слева** — пользователь может мгновенно переключиться на любой раздел
✅ **Горизонтальные табы вверху** — для внутренней навигации внутри раздела
✅ **Breadcrumbs (хлебные крошки)** — дополнительно показывают путь: Настройки → Клиника
✅ **Dropdown для большого количества табов** — кнопка "Ещё ▼" если табов больше 6-7

**Этот паттерн применяется ко всем разделам с подразделами:**
- Settings (10 табов)
- Аналитика (множество дашбордов)
- Отчёты (типы отчётов)
- Интеграции (категории интеграций)
- Карта пациента (вкладки внутри карты)

## Структура по ролям

### 1. Владелец клиники (Owner)

**Доступ ко всему + управление:**

```
✅ Главная (полный Dashboard с финансами)
✅ Расписание (просмотр всех врачей)
✅ Заявки (полный доступ, настройка автоматизации)
✅ Задачи (все задачи клиники, назначение задач любым сотрудникам)
✅ Пациенты
✅ Шаблоны (создание и редактирование общих и личных шаблонов)
✅ ОМС/ДМС (страховые компании, договоры, реестры)
✅ Лаборатории (заказы, финансы, статусы)
✅ Медкарта
✅ Финансы (все подразделы)
✅ Склад (номенклатура, остатки, автосписания)
✅ Аналитика (полная)
✅ Отчёты (все ведомости и реестры)
✅ Интеграции (полный доступ ко всем интеграциям)
✅ Настройки (полный доступ):
   • Клиника
   • Филиалы
   • Сотрудники
   • Подписка и оплата (только Owner)
   • Дополнительные возможности (только Owner)
   • Уведомления
   • Услуги
   • Виджет онлайн-записи
   • Документы и печать
   • Интеграции
```

## Детальная структура разделов

### 🏠 Главная (Dashboard)

**Подразделы:**
- Обзор дня (KPI, записи, задачи)
- Быстрые действия

**Роли:**
- Владелец: полные метрики + финансы
- Врач: своё расписание + свои пациенты
- Админ: операционные метрики

---

### 📅 Расписание

**Подразделы:**
- Календарь (день/неделя/месяц)
- Управление расписанием врачей
- Исключения (отпуска, больничные)

**Действия:**
- Создать запись
- Отменить/перенести
- Отметить статус (пришёл/не пришёл)

---

### 📥 Заявки (Входящие записи)

**Управление заявками на запись от пациентов из внешних источников**

**URL:** `/bookings` или `/requests`

**Доступ:** Owner, Admin (полный), Doctor (только свои заявки)

---

#### Источники заявок

Заявки могут поступать из разных каналов:

**1. Виджет на сайте клиники:**
- Форма онлайн-записи на вашем сайте
- Выбор врача, услуги, даты и времени
- Автоматическое определение свободных слотов

**2. Агрегаторы:**
- Яндекс.Карты (кнопка "Записаться")
- 2ГИС
- ПроДокторов.ру
- НаПоправку.ру

**3. Социальные сети и мессенджеры:**
- Instagram (через Direct)
- VK (через сообщения сообщества)
- WhatsApp Business
- Telegram Bot
- Viber

**4. Телефонные звонки:**
- Автоматическое создание заявки при входящем звонке (через IP-телефонию)
- Администратор вручную создаёт заявку по телефону

**5. Мобильное приложение для пациентов:**
- OrisiosPatient (запись через приложение)

#### Настройки автоматизации

**Три режима работы с заявками:**

##### 1. Ручной режим (по умолчанию)

```
📋 Ручная обработка всех заявок
├── Все заявки попадают в очередь "Новые"
├── Администратор вручную проверяет каждую заявку
├── Решает: подтвердить, изменить время или отклонить
└── Максимальный контроль, подходит для начинающих клиник
```

**Плюсы:**
- ✅ Полный контроль над расписанием
- ✅ Можно учесть особенности (врач задерживается, нужна предоплата)
- ✅ Возможность позвонить пациенту перед подтверждением

**Минусы:**
- ❌ Требует времени администратора
- ❌ Заявки могут ждать несколько часов

##### 2. Полуавтоматический режим (рекомендуется)

```
🤖 Умная обработка с проверкой
├── Система проверяет:
│   ├── Свободен ли слот?
│   ├── Работает ли врач в это время?
│   ├── Нет ли перерыва/обеда?
│   └── Соответствует ли специализация врача услуге?
│
├── Если всё ОК:
│   └── Заявка помечается ✅ (готова к подтверждению)
│       └── Администратор одним кликом подтверждает
│
└── Если проблема:
    └── Заявка помечается ⚠️ (требует внимания)
        └── Система предлагает альтернативные слоты
```

**Настройки полуавтоматического режима:**

- **Приоритет врачей:**
  - Записывать к конкретному врачу (если пациент выбрал)
  - ИЛИ к любому свободному врачу нужной специализации

- **Буферное время:**
  - Не записывать за X минут до закрытия (например, за 30 минут)
  - Минимальный интервал между записями (5-10 минут на подготовку)

- **Ограничения:**
  - Максимум новых пациентов в день (например, не более 5)
  - Требовать предоплату для дорогих услуг (от 10,000 ₽)

**Плюсы:**
- ✅ Быстрая обработка (пациент получает ответ в течение часа)
- ✅ Администратор тратит минимум времени
- ✅ Контроль остаётся у клиники

##### 3. Автоматический режим (для опытных клиник)

```
⚡ Мгновенное подтверждение
├── Система автоматически создаёт запись, если:
│   ├── ✅ Слот свободен
│   ├── ✅ Врач работает
│   ├── ✅ Нет перерыва/обеда
│   ├── ✅ Услуга соответствует специализации
│   └── ✅ Прошли все настроенные проверки
│
├── Пациент получает:
│   ├── SMS с подтверждением (сразу)
│   ├── Email с деталями записи
│   └── Ссылку для отмены/переноса
│
└── Администратор видит уведомление (не требует действий)
```

**Настройки автоматического режима:**

- **Правила автоподтверждения:**
  - ☑ Только для повторных пациентов (уже есть в базе)
  - ☑ Только для определённых услуг (консультация, чистка)
  - ☑ Только в рабочие часы (пн-пт 9:00-18:00)
  - ☑ Не для срочных случаев (например, "острая боль")

- **Временные ограничения:**
  - Не автоподтверждать запись менее чем за 2 часа (слишком срочно)
  - Не автоподтверждать запись более чем на 30 дней вперёд

- **Обязательные поля:**
  - Требовать указание телефона
  - Требовать указание email
  - Требовать выбор конкретной услуги

**Плюсы:**
- ✅ Мгновенное подтверждение (пациент доволен)
- ✅ Работает 24/7 (даже ночью и в выходные)
- ✅ Нет нагрузки на администратора

**Минусы:**
- ⚠️ Нужно тщательно настроить правила
- ⚠️ Возможны редкие ошибки (если расписание изменилось вручную)

### 📊 Аналитика

**Интерактивные дашборды и визуализация:**

**Финансовая аналитика:**
- График выручки (день/неделя/месяц/год)
- Воронка продаж (консультация → план → оплата)
- Средний чек
- Динамика прибыли по филиалам
- Структура выручки по услугам (диаграмма)

**Клиническая аналитика:**
- Загрузка врачей (визуализация по дням/неделям)
- Популярные услуги (топ-10 с графиками)
- Конверсия записей (записался → пришёл → повторный визит)
- Средняя длительность лечения
- Распределение диагнозов

**Маркетинговая аналитика:**
- Источники пациентов (откуда пришли)
- LTV (пожизненная ценность пациента)
- Retention rate (удержание пациентов)
- Эффективность акций
- Воронка из листа ожидания

**Пациенты:**
- Новые vs повторные
- Пациенты без визитов (> 6 месяцев)
- География пациентов
- Возрастная структура

**Складская аналитика:**
- Расход материалов по периодам
- ABC-анализ материалов
- Скорость оборачиваемости

**Расчёт зарплат врачей:**
- **Выручка по врачам:**
  - Таблица: Врач, Схема оплаты, Выручка, Приёмов, К выплате
  - График выручки по врачам (сравнение)
  - Детализация по услугам для каждого врача

- **Расчёт за период (например, за январь 2026):**
  - Выполненные и оплаченные услуги
  - Расчёт по схеме оплаты (оклад/процент/комбо)
  - KPI врача (если включено):
    - План vs Факт по приёмам
    - План vs Факт по выручке
    - Средний чек
    - Процент повторных визитов
  - Бонусы за выполнение KPI (если включено)
  - Вычеты и штрафы (если включено)
  - Итоговая сумма к выплате

- **Интерактивный калькулятор:**
  - Выбор врача
  - Выбор периода
  - Автоматический расчёт по настроенной схеме
  - Возможность ручной корректировки (например, добавить премию)

- **Сравнение периодов:**
  - Динамика зарплат по месяцам
  - Изменение выручки врачей
  - Эффективность работы (выручка на 1 приём)

- **Кнопка:** "Сформировать ведомость" → переход в Отчёты

**Особенности:**
- Все графики интерактивные
- Фильтры по датам, филиалам, врачам
- Экспорт в Excel/PDF
- Сравнение периодов

### 📋 Отчёты (Ведомости и реестры)

**Табличные документы для печати, экспорта и проверки:**

**Финансовые отчёты:**
- **Реестр счетов** (все счета за период с фильтрами)
  - По статусу (ожидает/оплачен/частично)
  - По врачу, пациенту, услуге
  - С суммами и датами
- **Реестр оплат** (журнал всех платежей)
  - Наличные/безналичные/смешанные
  - По кассирам
  - По способу оплаты
- **Отчёт по выручке** (детальный за период)
  - По услугам
  - По врачам
  - По филиалам
- **Задолженности пациентов** (кто и сколько должен)
- **Отчёт по услугам** (какие услуги оказаны, количество, сумма)
- **Акты выполненных работ** (для страховых компаний)

**Клинические отчёты:**
- **Реестр пациентов** (полный список с контактами)
  - С фильтрами по дате регистрации
  - По врачу
  - По статусу (активный/неактивный)
- **Журнал приёмов** (все визиты за период)
  - Дата, время, врач, пациент, услуга
  - Статус приёма (состоялся/не пришёл)
- **Реестр медкарт** (список всех медкарт)
- **Статистика по диагнозам** (какие диагнозы ставились)
- **Отчёт по гарантиям** (истекающие гарантийные сроки)

**Складские отчёты:**
- **Реестр материалов** (остатки на складе)
  - С указанием минимумов
  - С выделением критических остатков
- **Отчёт по списаниям** (что и когда списали)
  - По врачам
  - По пациентам
  - По материалам
- **Инвентаризация** (сверка остатков)
- **Отчёт по поступлениям** (закупки)

**Кадровые отчёты:**
- **Загрузка врачей** (табличный с часами работы)
- **Выполненные услуги по врачам** (что сделал каждый врач)
- **Отчёт по рабочему времени**
- **KPI сотрудников**

**Маркетинговые отчёты:**
- **Источники пациентов** (откуда пришли: реклама, сайт, рекомендации)
- **Эффективность акций** (сколько привлекли пациентов)
- **Отчёт по программам лояльности** (начисления/списания бонусов)
- **Лист ожидания** (кто в листе, статус)

**Для внешних организаций:**
- **Реестр для ОМС/ДМС** (для страховых компаний)
- **Отчёты для налоговой** (выручка, услуги)
- **Реестр для лабораторий** (заказы, статусы)

**Особенности раздела:**
- Все отчёты с фильтрами (даты, филиалы, врачи, пациенты)
- Экспорт в Excel, PDF, CSV
- Печать с настройкой формата
- Шаблоны отчётов (можно сохранить фильтры)
- История формирования отчётов
- Автоматическая отправка по расписанию (опционально)

  - ✅ Готов к выдаче
  - 📦 Получен клиникой

  - Таблица заказов с суммами

### Технические детали

**Backend:**
- **WebSockets:** Socket.io или Pusher для real-time
- **База данных:**
  - `chats`: id, type (direct/group), clinic_id, created_at
  - `chat_members`: chat_id, user_id, last_read_at
  - `messages`: id, chat_id, sender_id, message_text, attachments, created_at, read_by[]
  - `chat_files`: id, message_id, file_name, file_url, file_size, mime_type

**Приватность:**
- Multi-tenancy: чаты изолированы по clinic_id (RLS)
- Сотрудники видят только чаты своей клиники
- При увольнении сотрудника — удаление из всех чатов, но сообщения остаются

**Хранение файлов:**
- AWS S3 / Cloudflare R2 / собственное хранилище
- CDN для быстрой загрузки изображений
- Автоматическое сжатие изображений (max 1920px)

**Производительность:**
- Lazy loading: подгрузка сообщений по мере прокрутки
- Virtual scrolling для длинных переписок
- Debounce для "печатает..." (500ms)

**Уведомления:**
- Browser Push API (если разрешено)
- In-app notifications (badge + звук)
- Email digest (опционально, 1 раз в день с пропущенными сообщениями)

### Технические детали

**База данных:**

**Существующие поля (таблица `patients`):**
- `date_of_birth` (date) — дата рождения пациента
- `phone` (varchar) — телефон
- `email` (varchar) — email
- `clinic_id` (uuid) — привязка к клинике

**Новые поля (опционально, для Phase 2+):**
- `birthday_greeting_sent_at` (timestamp) — когда последний раз поздравляли
- `birthday_greeting_channel` (enum: 'sms', 'email', 'call', 'whatsapp') — каким способом
- `birthday_greeting_count` (int) — сколько раз поздравляли всего
- `birthday_discount_applied` (boolean) — применялась ли скидка в этом году

**SQL запрос для виджета (3 дня вперёд):**

```sql
SELECT
  id,
  first_name,
  last_name,
  date_of_birth,
  phone,
  EXTRACT(YEAR FROM AGE(CURRENT_DATE, date_of_birth)) + 1 AS turning_age,
  CASE
    WHEN EXTRACT(DAY FROM date_of_birth) = EXTRACT(DAY FROM CURRENT_DATE)
      AND EXTRACT(MONTH FROM date_of_birth) = EXTRACT(MONTH FROM CURRENT_DATE)
      THEN 0  -- Сегодня
    WHEN EXTRACT(DAY FROM date_of_birth) = EXTRACT(DAY FROM CURRENT_DATE + INTERVAL '1 day')
      AND EXTRACT(MONTH FROM date_of_birth) = EXTRACT(MONTH FROM CURRENT_DATE + INTERVAL '1 day')
      THEN 1  -- Завтра
    WHEN EXTRACT(DAY FROM date_of_birth) = EXTRACT(DAY FROM CURRENT_DATE + INTERVAL '2 days')
      AND EXTRACT(MONTH FROM date_of_birth) = EXTRACT(MONTH FROM CURRENT_DATE + INTERVAL '2 days')
      THEN 2  -- Через 2 дня
  END AS days_until
FROM patients
WHERE clinic_id = :clinic_id
  AND (
    (EXTRACT(MONTH FROM date_of_birth) = EXTRACT(MONTH FROM CURRENT_DATE)
     AND EXTRACT(DAY FROM date_of_birth) BETWEEN EXTRACT(DAY FROM CURRENT_DATE)
       AND EXTRACT(DAY FROM CURRENT_DATE + INTERVAL '2 days'))
    OR
    (EXTRACT(MONTH FROM date_of_birth) = EXTRACT(MONTH FROM CURRENT_DATE + INTERVAL '1 day')
     AND EXTRACT(DAY FROM date_of_birth) = EXTRACT(DAY FROM CURRENT_DATE + INTERVAL '1 day'))
    OR
    (EXTRACT(MONTH FROM date_of_birth) = EXTRACT(MONTH FROM CURRENT_DATE + INTERVAL '2 days')
     AND EXTRACT(DAY FROM date_of_birth) = EXTRACT(DAY FROM CURRENT_DATE + INTERVAL '2 days'))
  )
ORDER BY days_until ASC, date_of_birth ASC
LIMIT 10;
```

**Автоматическое уведомление (Cron job):**
- Каждый день в 09:00 (настраивается)
- Проверка: есть ли пациенты с ДР сегодня
- Если да → создать уведомление для всех сотрудников клиники
- Отправка в таблицу `notifications`

**SMS-интеграция:**
- Использование существующей SMS-интеграции (SMS.ru / SMSC.ru)
- API вызов при клике на ✉️ кнопку
- Шаблоны хранятся в таблице `sms_templates` (type: 'birthday')

**Приватность:**
- Multi-tenancy: пациенты изолированы по `clinic_id`
- RLS (Row Level Security) на уровне базы данных
- Сотрудники видят ДР только пациентов своей клиники
