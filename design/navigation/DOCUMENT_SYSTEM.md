# Система документов и печати Orisios

> Единый центр управления документами пациента с настраиваемыми пакетами и гибкой печатью

---

## Концепция

**Единое место для всех документов пациента** — карта пациента → вкладка "📄 Документы"

### Принципы работы:

✅ **Быстрые пакеты** — печать 3-6 документов одной кнопкой (настраиваемые)
✅ **Отдельные документы** — печать любого документа в любой момент
✅ **Произвольный выбор** — любая комбинация документов
✅ **Автосохранение** — всё попадает в архив автоматически
✅ **Повторная печать** — из архива за 1 клик

## Типы документов в стоматологии

### 📋 Договорные документы (до лечения)

| Документ | Назначение | Требует подпись |
|----------|------------|-----------------|
| Договор на оказание услуг | Основной договор с клиникой | ✅ Да |
| Информированное согласие | Общее согласие на лечение | ✅ Да |
| Согласие на хирургическое вмешательство | Для операций, имплантации | ✅ Да |
| Согласие на анестезию | Отдельное согласие на наркоз | ✅ Да |
| Согласие на обработку ПД | 152-ФЗ, обработка персональных данных | ✅ Да |
| Согласие родителей | Для детей до 15 лет | ✅ Да |
| Анкета здоровья (анамнез) | Сбор медицинской информации | ❌ Нет |

### 🏥 Клинические документы

| Документ | Назначение | Требует подпись |
|----------|------------|-----------------|
| План лечения | Описание предстоящего лечения | ❌ Нет |
| Выписка из медкарты | Краткая история лечения | ❌ Нет |
| Дневник врача | Подробное описание приёма | ❌ Нет |
| Рекомендации после лечения | Инструкции для пациента | ❌ Нет |
| Рекомендации после имплантации | Специальные инструкции 7-14 дней | ❌ Нет |
| Рекомендации после удаления | Уход после хирургии | ❌ Нет |
| Гарантийный талон | Гарантия на работы | ✅ Да |
| Направление в лабораторию | Для зуботехнической работы | ❌ Нет |
| Справка для работы | Подтверждение посещения врача | ❌ Нет |
| Справка для учебы | Для студентов | ❌ Нет |

### 💰 Финансовые документы

| Документ | Назначение | Требует подпись |
|----------|------------|-----------------|
| Счёт на оплату | Перечень услуг и сумма | ❌ Нет |
| Квитанция об оплате | Подтверждение оплаты | ❌ Нет |
| Акт выполненных работ | Закрывающий документ | ✅ Да |
| Договор рассрочки | Условия рассрочки платежа | ✅ Да |

## Режимы работы

### 1. 🚀 Быстрые пакеты (типовые сценарии)

**Назначение:** Печать стандартного набора документов одной кнопкой

**Системные пакеты (встроенные, настраиваемые):**

#### 📋 Первичный приём (4 документа)
```
1. Договор на оказание услуг
2. Информированное согласие на лечение
3. Согласие на обработку персональных данных
4. Анкета здоровья

[🖨 Печать пакета (4)]
```

#### 🦷 Завершение лечения (3 документа)
```
1. Рекомендации после лечения
2. Выписка из медицинской карты
3. Гарантийный талон

[🖨 Печать пакета (3)]
```

#### 💰 Финансовые документы (2 документа)
```
1. Счёт на оплату
2. Договор рассрочки (если применимо)

[🖨 Печать пакета]
```

**Пользовательские пакеты (созданные клиникой):**

Примеры:
- 🦷 Имплантация (6 документов)
- 🔬 Ортодонтия (5 документов)
- 📄 Справки и выписки (2 документа)

---

### Метаданные каждого документа:

- Дата и время создания
- Кто создал (пользователь)
- Статус (подписан/не подписан)
- Связанный приём (если есть)
- Версия документа

### Действия с документами:

- 👁 **Просмотр** — открыть PDF в новой вкладке
- 🖨 **Повторная печать** — печать копии
- ⬇ **Скачать PDF** — сохранить на компьютер
- 📧 **Отправить на email** — отправить пациенту
- ✏️ **Редактировать** — изменить данные (если не подписан)
- 🗑️ **Удалить** — удалить из архива (с подтверждением)

## Сценарии использования

### Сценарий 1: Первичный приём пациента

```
Администратор записывает нового пациента:

1. Создал запись в расписании
2. Открыл карту пациента → 📄 Документы
3. Клик на [📋 Первичный приём]
4. Предпросмотр 4 документов
5. [🖨 Печать пакета]
6. Принтер печатает:
   • Договор (автозаполнен ФИО, дата, №)
   • Информированное согласие
   • Согласие ПДРФ
   • Анкета здоровья (бланк)
7. Передаёт пациенту на подпись
8. Все документы автоматически в архиве

Время: 30 секунд
```

### Сценарий 2: Завершение приёма

```
Врач завершил лечение:

1. Карта пациента → 📄 Документы
2. Клик на [🦷 Завершение лечения]
3. [🖨 Печать пакета]
4. Принтер печатает:
   • Рекомендации (автозаполнены из приёма)
   • Выписка из медкарты
   • Гарантийный талон
5. Пациент получает документы

Время: 20 секунд
```

### Сценарий 3: Нужна только справка

```
Пациент просит справку для работы:

1. Карта пациента → 📄 Документы
2. Раздел "🏥 КЛИНИЧЕСКИЕ" → [Развернуть]
3. Клик на [🖨 Печать] напротив "Справка для работы"
4. Заполнение полей (дата, врач, причина)
5. [🖨 Печать]
6. Готово

Время: 15 секунд
```

### Сценарий 4: Нестандартный набор

```
Пациенту нужно несколько разных документов:

1. Карта пациента → 📄 Документы
2. [📝 Произвольный выбор...]
3. Выбрал галочками:
   ☑ Договор
   ☑ План лечения
   ☑ Справка для работы
   ☑ Квитанция
   ☑ Гарантийный талон
4. [🖨 Печать (5)]
5. Опционально: ☑ Сохранить как пакет "Мой набор"
6. Готово

Время: 40 секунд
```

### Сценарий 5: Повторная печать

```
Пациент потерял договор:

1. Карта пациента → 📄 Документы
2. Архив → 📋 ДОГОВОРНЫЕ → [Развернуть]
3. Найти "Договор №45"
4. [🖨 Повторная печать]
5. Готово

Время: 10 секунд
```

## Преимущества системы

✅ **Скорость** — готовые пакеты печатают 3-6 документов за 1 клик
✅ **Гибкость** — можно печатать любой документ отдельно
✅ **Настраиваемость** — каждая клиника создаёт свои пакеты
✅ **Стандартизация** — все сотрудники печатают одинаково
✅ **История** — все документы в архиве с метаданными
✅ **Контроль** — видно кто, когда и что печатал
✅ **Повторная печать** — из архива за 1 клик
✅ **Масштабируемость** — неограниченное количество пакетов
